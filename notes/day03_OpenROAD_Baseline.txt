# 3. ASIC Physical Design Flow Understanding

The OpenROAD flow converts RTL Verilog into a physically manufacturable ASIC layout.

The major stages are:

| Stage           | Description                            |
| --------------- | -------------------------------------- |
| Synthesis       | Converts RTL into gate-level netlist   |
| Floorplan       | Defines chip core area and utilization |
| Placement       | Places standard cells physically       |
| CTS             | Builds clock distribution tree         |
| Routing         | Connects wires on metal layers         |
| Timing Analysis | Evaluates setup/hold timing            |
| GDS Generation  | Produces final layout database         |

# 5. Timing Analysis Concepts Learned

## Slack

Core timing formula:

```text
Slack = Required Time - Arrival Time
```

Meaning:

* Slack > 0 → timing met
* Slack < 0 → timing violation

Example:

```text
Slack = -0.03 ns
```

This means the path arrives 30 ps too late.

---

## WNS (Worst Negative Slack)

Formula:

```text
WNS = minimum slack among all paths
```

Observed result:

```text
WNS = -0.03 ns
```

Meaning:

The worst path violates timing by 30 ps.

---

## TNS (Total Negative Slack)

Formula:

```text
TNS = sum of all negative slacks
```

Observed result:

```text
TNS = -0.30 ns
```

Meaning:

All violating paths combined miss timing by 0.30 ns.

---

## Setup Violations

Observed:

```text
setup violation count = 11
```

Meaning:

11 timing paths fail setup timing.

---

## Hold Violations

Observed:

```text

```

Meaning:

No hold timing failures exist.

# 6. Timing Evolution Through the Flow

Observed timing trend:

| Stage            | WNS      |
| ---------------- | -------- |
| Floorplan        | +0.05 ns |
| Global Placement | +0.01 ns |
| CTS              | -0.03 ns |
| Routing          | -0.04 ns |
| Final            | -0.03 ns |

---

## Important Insight

Timing became worse after routing.

Reason:

Before routing:

* wire length is estimated
* parasitic RC is approximate

After routing:

* real wire geometry exists
* real resistance and capacitance are extracted
* delay increases

Therefore:

```text
Routing parasitics degrade timing.
```

This is one of the core problems in timing closure research.

---

# 7. Physical Meaning of Key Metrics

## Capacitance

Represents:

* wire capacitance
* gate input capacitance
* coupling capacitance

Large capacitance causes:

* larger delay
* slower transitions
* timing degradation

---

## Slew

Represents signal transition speed.

Poor slew means:

* slower rise/fall edges
* degraded timing
* higher power/noise

---

## Fanout

Represents how many loads one gate drives.

Larger fanout:

* increases effective capacitance
* increases delay
* worsens slack
